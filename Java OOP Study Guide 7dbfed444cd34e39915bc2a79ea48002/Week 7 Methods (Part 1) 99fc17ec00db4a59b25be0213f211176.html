Methods

1. Modules Approach

  • Definition: Modularizing a program by dividing tasks into self-contained methods.
  • Benefits:
    1. Simplifies code (Divide and Conquer).
    1. Increases reusability by using methods as building blocks.
    1. Makes programs easier to manage and debug.

Example (Sum Calculation Without Methods):

int sum = 0;
	for (int i = 1; i <= 10; i++) sum += i;
	System.out.println("Sum from 1 to 10 is " + sum);

With Methods:

public static int sum(int i1, int i2) {
		int sum = 0;
		for (int i = i1; i <= i2; i++) sum += i;
		return sum;
	}
	
	public static void main(String[] args) {
		System.out.println("Sum from 1 to 10 is " + sum(1, 10));
		System.out.println("Sum from 20 to 30 is " + sum(20, 30));
	}

2. Declaring Methods

  • Syntax:
    public static int methodName(int a, int b) {
		// Body of the method
	}
  • Components:
    1. public static: Modifiers (to be explained).
    1. int: Return type (e.g., int, void).
    1. methodName: Method's name (should start with a lowercase verb).
    1. (int a, int b): Parameters (data passed into the method).
  • Types of Methods:
    1. Predefined Methods: Already defined in Java (e.g., Math.max()).
      System.out.println(Math.max(9, 7)); // Output: 9
    1. User-defined Methods: Created by the programmer.
      public static void findEvenOdd(int num) {
		if (num % 2 == 0) System.out.println(num + " is even");
		else System.out.println(num + " is odd");
	}
  • Return Values:
    • Use return to send a value back to the caller.
      public static int add(int a, int b) {
		return a + b;
	}

3. Arguments and Casting

  • Passing Parameters to Methods:
    • Parameters are passed by value (a copy of the variable is passed).
      public static void main(String[] args) {
		int number = 1;
		tryToChangeNumber(number);
		System.out.println(number); // Output: 1 (value not changed)
	}
	
	public static void tryToChangeNumber(int i) {
		i = 2; // Only changes the local copy
	}
  • Multiple Parameters:
    public static void myMethod(String fname, int age) {
		System.out.println(fname + " is " + age);
	}
	
	public static void main(String[] args) {
		myMethod("Liam", 5);
		myMethod("Jenny", 8);
	}
	// Output:
	// Liam is 5
	// Jenny is 8
  • Argument Promotion:
    • Java can promote smaller data types to larger ones when required.
      public static int square(int x) {
		return x * x;
	}
	
	public static void main(String[] args) {
		double value = 4.5;
		System.out.println(square((int) value)); // Output: 16
	}

4. Java API Packages

  • Definition:
    • Packages group related classes into categories for better organization and reuse.
    • Example: javax.swing.JApplet.
  • Benefits:
    1. Promotes software reuse.
    1. Simplifies class access through the import statement.
    import java.util.Scanner;  // Importing Scanner class
	public class Example {
		public static void main(String[] args) {
			Scanner input = new Scanner(System.in);
			System.out.println("Enter a number: ");
			int number = input.nextInt();
			System.out.println("You entered: " + number);
		}

5. Scope of Declaration

  • Definition: Scope refers to the portion of a program where a declared entity (e.g., variable, method) can be referenced by its name.
  • Types of Scope:
    1. Parameter Declaration Scope: Exists within the method in which the parameter is declared.
    1. Local Variable Scope: Limited to the block or method where the variable is declared.
    1. Class Member Scope: Accessible by all methods within the class.
    public class ScopeExample {
		public static void main(String[] args) {
			int x = 10; // Local variable
			if (x > 5) {
				int y = 20; // Block scope
				System.out.println("x + y = " + (x + y));
			}
			System.out.println(y); // Error: y is out of scope
		}
	}

6. Method Overloading

  • Definition: Allows multiple methods with the same name but different parameter lists.
  • Key Points:
    1. Methods must have different numbers or types of parameters.
    1. Overloading does not depend on the return type.
  • Example 1: Overloading with Different Parameter Types
    class Adder {
		static int add(int a, int b) {
			return a + b;
		}
	
		static double add(double a, double b) {
			return a + b;
		}
	
		public static void main(String[] args) {
			System.out.println(add(10, 20));
			// Output: 30
			
			System.out.println(add(10.5, 20.5));
			// Output: 31.0
		}
	}
  • Example 2: Overloading with Different Numbers of Parameters
    class Adder {
		static int add(int a, int b) {
			return a + b;
		}
	
		static int add(int a, int b, int c) {
			return a + b + c;
		}
	
		public static void main(String[] args) {
			System.out.println(add(10, 20));
			// Output: 30
			
			System.out.println(add(10, 20, 30));
			// Output: 60
		}
	}
  • Example 3: Overloading with Type Promotion
    class OverloadingExample {
		static void sum(int a, long b) {
			System.out.println(a + b);
		}
	
		static void sum(int a, int b, int c) {
			System.out.println(a + b + c);
		}
	
		public static void main(String[] args) {
			sum(20, 30L);
			// Output: 50 (int promoted to long)
			
			sum(10, 20, 30);
			// Output: 60
		}
	}
