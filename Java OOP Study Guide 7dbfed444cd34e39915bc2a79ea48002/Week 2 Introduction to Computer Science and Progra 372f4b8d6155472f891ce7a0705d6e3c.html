Introduction to Hardware and Software:

  • Hardware: Physical components like the mouse, keyboard, RAM, and CPU.
  • Software: Programs running on hardware, controlled by sequences of instructions called "computer programs."
  • Java: A widely-used programming language designed for desktop, server, and web-based applications.

Computer Organization:

  • Logical Units of a Computer System:
    1. Input Unit: Mouse, keyboard.
    1. Output Unit: Printer, monitor, audio speakers.
    1. Memory Unit: RAM.
    1. Arithmetic and Logic Unit (ALU): Performs calculations.
    1. CPU: Supervises device operations.
    1. Secondary Storage: Hard drives, floppy drives.

Programming Languages:

Low-Level Programming:

  • Machine language: Hardware-specific binary instructions.
  • Assembly language: Simplified English-like codes.

High-Level Programming:

  • Use compilers to translate high-level languages into machine language.
  • Interpreters execute programs directly, avoiding compilation delays.

Operating Systems:

  • Facilitate efficient, safe, and parallel execution of applications.
  • Core component: Kernel
  • Examples: Windows, Linux, macOS, Android.

Java Development Environment:

  • Phase 1: Creating a Program
    • Write source code in .java files using an editor or IDE.
    • Popular IDEs: Eclipse, NetBeans, IntelliJ IDEA, BlueJ.
  • Phase 2: Compiling
    • Use javac (the Java compiler) to compile .java files into bytecodes (.class files).

  • Phase 3: Loading
    • JVM loads the program into memory.
    • The JVM’s class loader takes the .class files containing the program’s bytecodes and transfers them to primary memory.
  • Phase 4: Bytecode Verification
    • Ensures bytecodes are valid and secure.
  • Phase 5: Execution
    • JVM executes bytecodes using interpretation and just-in-time (JIT) compilation.
    • One in which source code is translated into bytecodes.
    • Second in which, during execution, the bytecodes are translated into machine language for the actual computer on which the program executes
  • Common Programming Error
    • Errors such as division by zero occur as a program runs, so they’re called runtime errors or execution-time errors.
    • Fatal runtime errors cause programs to terminate immediately without having successfully performed their jobs.
    • Nonfatal runtime errors allow programs to run to completion, often producing incorrect results.

Internet and World Wide Web:

  • Internet: Initially developed for connecting a few mainframe systems. Now connects millions of computers worldwide.
  • WWW: Enables locating and viewing multimedia-based documents.

Introduction to Object Technology:

  • Objects: Reusable software components modeling real-world items.
    • Attributes: Size, shape, color.
    • Behaviors: Actions like making calls or stopping engines.
  • Object-Oriented Design (OOD):
    • Encapsulates data and behaviors.
    • Communicates through well-defined interfaces.
    • Programming language: Java.
