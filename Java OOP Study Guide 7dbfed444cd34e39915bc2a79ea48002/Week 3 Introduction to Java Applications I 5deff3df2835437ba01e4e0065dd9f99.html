Introduction to Java Applications

1. Writing First Program in Java

  • Class Structure:
    • Class Declaration: public class Welcome1
    • Method Declaration: public static void main(String[] args)
  • Compiling and Running:
    • Compile: javac Welcome1.java
    • Execute: java Welcome1
  • Comments:
    • Single-line: //
    • Multi-line: /* ... */
    • Javadoc: /** ... */

Blank Line

  • Makes the program more readable.
  • Blank lines, spaces, and tabs are white-space characters.
  • Ignored by the compiler.

Begins Class Declaration for Class Welcome1

  • Every Java program has at least one user-defined class.
  • Keyword: Words reserved for use by Java.
  • Class Declaration Syntax: Use the class keyword followed by the class name.

Name of Class Called Identifier

  • Definition:

    An identifier is the name of a class in Java.

  • Rules for Identifiers:
    • Consists of letters, digits, underscores (_), and dollar signs ($).
    • Cannot start with a digit or contain spaces.
  • Examples:
    • Valid: Welcome1, $value, _value, button7
    • Invalid: 7button
  • Case Sensitivity:
    • Java is case-sensitive.
    • Examples: a1 and A1 are considered different.

Saving Files

  • The file name must match the class name and have the .java extension.
  • Example: Welcome1.java.

Left Brace {

  • Purpose:
    • Begins the body of every class.
    • The right brace } ends declarations

Part of Every Java Application

  1. Applications begin executing at the main method.
  1. Parentheses () indicate that main is a method (discussed further in Chapter 6).
  1. Methods: Java applications contain one or more methods.

Instructs Computer to Perform an Action

  • Purpose: Prints a string of characters.
    • A string is a series of characters enclosed within double quotes " ".
    • White-spaces in strings are not ignored by the compiler.

System.out

  • Definition:
    • Standard output object.
    • Prints output to the command window (e.g., MS-DOS prompt).

Method System.out.println

  • Purpose: Displays a line of text.
  • Arguments: Content inside parentheses ().

Example:

System.out.println("Hello, World!");

This Line Known as a Statement

  • Statements must end with a semicolon (;).

Common Escape Sequences in Java

Escape SequenceDescription
\nNewline: Moves the cursor to the beginning of the next line.
\tHorizontal Tab: Moves the cursor to the next tab stop.
\rCarriage Return: Moves the cursor to the beginning of the current line.

2. Displaying Text

  • Methods:
    • print - Prints text without newline.
    • println - Prints text with a newline.
    • printf - Used for formatted text output.
  • Format Specifiers:
    • %s - Strings
    • %d - Integers
    • %f - Floating-point numbers
    • %n - Newline
  • Example:
    System.out.printf("Hello %s!%n", "World");

    Output: Hello World!

  • 1. Boolean Formatting:

    Purpose: Formats and displays boolean values. ُ

    System.out.printf("%b%n", null);        // Output: false
	System.out.printf("%B%n", false);       // Output: FALSE
	System.out.printf("%B%n", 5.3);         // Output: TRUE
	System.out.printf("%b%n", "random text");// Output: true
  • 2. String Formatting:

    Purpose: Formats strings with optional uppercase conversion and field width.

    printf("'%s' %n", "baeldung");
	// Output: 'baeldung'
	printf("'%S' %n", "baeldung");
	// Output: 'BAELDUNG'
	printf("'%15s' %n", "baeldung");
	// Output: '       baeldung'
  • 3. Number Formatting:

    Purpose: Formats numbers with grouping and precision. 

    System.out.printf(Locale.US, "%,d %n", 10000);
	// Output: 10,000
	System.out.printf(Locale.ITALY, "%,d %n", 10000);
	// Output: 10.000
	System.out.printf("'%5.2f'%n", 5.1473);          // Output: ' 5.15'
  • 4. Date and Time Formatting:

    Purpose: Formats date and time using specific placeholders.

    System.out.printf("hours %tH: minutes %tM: seconds %tS%n", date, date, date);
	// Output: hours 13: minutes 51: seconds 15
	System.out.printf("%1$tA, %1$tB %1$tY %n", date);
	// Output: Thursday, November 2018

Key Notes:

  • %b / %B : Boolean values (lowercase/uppercase).
  • %s / %S : Strings (lowercase/uppercase).
  • %d : Integers with grouping (Locale).
  • %f : Floating-point numbers (precision).
  • %tH, %tM, %tS : Hours, Minutes, Seconds.
  • %tA, %tB, %tY : Day, Month, Year.

3. Obtaining User Input

  • Using Scanner Class:
    • Import: import java.util.Scanner;
    • Create Scanner: Scanner input = new Scanner(System.in);
  • Reading Inputs:
    • Integer: int number = input.nextInt();
    • Double: double value = input.nextDouble();
    • String: String text = input.next();
    • Line: String line = input.nextLine();

import java.util.Scanner; // Import the Scanner class 
	class Main { 
		public static void main(String[] args) { 
			Scanner input = new Scanner(System.in); // Create a Scanner object 
			System.out.println("Enter username"); 
			String userName = input.nextLine(); // Read a line 
			
			System.out.println("Enter your age"); 
			int userAge = input.nextInt(); // Read an integer
			
			System.out.println("Enter user weight"); 
			double userWeight = input.nextDouble(); // Read a double 
			
			System.out.println("User name is: " + userName); // Output user input 
			System.out.println("User age is: " + userAge); // Output user input 
			System.out.println("Username is: " + userWeight); // Output user input 
		}
	}

6. Debugging and Tracing Errors

  • Debugging: is the routine process of locating and removing bugs, errors or abnormalities from programs.
  • Breakpoints: Pause program execution to inspect variables.
  • Exception Breakpoints: Detect runtime exceptions like NullPointerException.
  • Watchpoints: Track when a variable's value changes.
  • Debugging Tools: Use debugging features in NetBeans IDE.

7. Variables

  • Definition: Location in memory that stores a value.
  • Rules:
    • Name must be a valid identifier (letters, digits, _, $).
    • Cannot begin with a digit or contain spaces.
    • Java is case-sensitive (a1 ≠ A1).
int number1; // Declare variable
	int number1, number2; // Multiple variables
	int myNum = 15; // Declare and assign
	final int CONSTANT = 10; // Final variable (read-only)

Primitive Data Types in Java (Summary):

  • Integers
    • Store whole numbers (e.g., 72, 1127, 0).

  • Float & Double
    • Store real numbers with decimals (e.g., 3.4, 0.0, 11.19).
  • Char
    • Store single characters (e.g., A, 7, , %).
  • Boolean
    • Store True or False values.

Note: There are eight primitive types: boolean, byte, char, short, int, long, float, and double.

Final Variables in Java

  • Declaring a variable as final makes it constant (unchangeable and read-only).
  • Once assigned, its value cannot be modified.
final int myNum = 10;
	myNum = 20; // Error: cannot assign a value to a final variable

8. Memory Concepts

  • Variables refer to memory locations.
  • Assigning a value replaces the previous content in that location.
int number1 = 45;
	number1 = 72; // Replaces value 45 with 72

9. Operators

  • Arithmetic Operators:
    • Addition (+), Subtraction (-), Multiplication (*), Division (/), Remainder (%).
    • Integer division truncates the remainder.
int result = 7 / 5; // result = 1
	int remainder = 7 % 5; // remainder = 2
  • Precedence: Parentheses control the order of operations.
    • Multiplication, division, and remainder first (left-to-right).
    • Addition and subtraction next.

Equality Operators:

  1. == : Checks if two values are equal. | Example: x == y → x is equal to y.
  1. != : Checks if two values are not equal. | Example: x != y → x is not equal to y.

Relational Operators:

  1. > : Checks if x is greater than y. | Example: x > y.
  1. < : Checks if x is less than y. | Example: x < y.
  1. >= : Checks if x is greater than or equal to y. | Example: x >= y.
  1. <= : Checks if x is less than or equal to y. | Example: x <= y.

10. Increment/Decrement Operators

  • Pre-Increment (++x): Increments before use.
  • Post-Increment (x++): Increments after use.
int b = 15, a;
	
	// Pre-Increment
	a = ++b; // a = 16
	
	// Post-Increment
	a = b++; // a = 15
  • Similarly, - for decrement.

11. Assignment & Expressions

  • Assignment Operator (=): Assigns a value.
  • Compound Assignment (+=, =, etc.): Modifies and assigns.
int x = 10;
	x += 5; // x = x + 5 => x = 15

12. Logic and Syntax Errors

  • Syntax Errors: Detected by the compiler (e.g., missing semicolon, parentheses).
System.out.print // Missing parentheses
	int x = 10 // Missing semicolon
  • Logic Errors: Program compiles and runs but produces incorrect results.

Incorrect operator precedence:

int result = 5 + 4 * 3 / 2; // Incorrect result

13. Debugging Concepts

  • Runtime Errors: Errors during program execution (e.g., divide by zero).
    • Example:
    int result = 15 / 0; // Runtime error
  • Debugging tools help identify and fix errors.
