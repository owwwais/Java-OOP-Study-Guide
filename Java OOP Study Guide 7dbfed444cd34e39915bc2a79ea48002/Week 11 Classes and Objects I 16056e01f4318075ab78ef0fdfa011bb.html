Classes and Objects

1. Declaring Classes, Instance Variables, and Methods

Class: A blueprint for creating objects.

  • Contains fields, methods, constructors, Blocks, and nested classes.
    class Lamp {
		boolean isOn; // Instance variable
		void turnOn() { isOn = true; }
		void turnOff() { isOn = false; }
	}

Object: is an instance of a class, created from a blueprint (class).

Lamp led = new Lamp(); // Create object
	led.turnOn();

Instance Variables:

  • Declared inside a class but outside any method.
  • Memory is allocated when an object is created.
public class Student {
		int rollNo; // Instance variable
	}

Access Modifiers: public and private

  • public:
    • Variables or methods declared as public are accessible from outside the class.
    • Example:
      public class MyClass {
		public int number = 10;
		public void display() {
			System.out.println(number);
		}
	}
  • private:
    • Variables or methods declared as private are accessible only within the class where they’re declared.
    • Example:
      public class MyClass {
		private int number = 10;
		private void display() {
			System.out.println(number);
		}
	}
  • public: Accessible everywhere.
  • private: Accessible only within the class.

2. Default and Explicit Initialization for Instance Variables

  • Default Initialization:
    • Primitive types:
      • int, long, float → 0
      • boolean → false
    • Reference types → null
  • Explicit Initialization:
    class MyClass {
		int x = 10; // Explicit initialization
	}

3. Constructors

  • Definition: Special methods to initialize objects.
    • Same name as the class, no return type.
      class MyClass {
		int x;
		MyClass(int value) { x = value; } // Constructor
	}
  • Parameterized Constructor Example:
    MyClass obj = new MyClass(100); // Passing argument to constructor
  • Static Variables and Methods:
    • Static Variables: Shared among all objects.
    • Static Methods: Associated with the class, not objects.
      class Calculate {
		static int cube(int x) { return x * x * x; }
	}

Access Levels for Modifiers

  • Default: Accessible within the same package (subclass & non-subclass).
  • Private: Accessible only within the same class.
  • Protected: Accessible in the same package or subclass (even in a different package).
  • Public: Accessible from anywhere.

Key Takeaways:

  • Public: Use for global access.
  • Private: Use for encapsulation and restricting access.
  • Protected: Use for inheritance with controlled access.
  • Default: Use for internal package access.

4. Instantiating and Using Objects

  • Objects are created using the new keyword:
    Lamp led = new Lamp(); // Object of Lamp class
	led.turnOn(); // Call method
  • Access Modifiers:
    • public: Accessible everywhere.
    • private: Accessible only within the class.

5. Implementing a Test Class

  • Example 1: Lamp Class
    class Lamp {
		boolean isOn;
		void turnOn() { isOn = true; System.out.println("Light on? " + isOn); }
		void turnOff() { isOn = false; System.out.println("Light on? " + isOn); }
	}
	public class Main {
		public static void main(String[] args) {
			Lamp led = new Lamp();
			led.turnOn();
		}
	}

    Output:

    Light on? true
  • Example 2: Account Class
    class Account {
		int accNo;
		String name;
		float balance;
	
		void insert(int accNo, String name, float balance) {
			this.accNo = accNo;
			this.name = name;
			this.balance = balance;
		}
	}

6. this Reference

  • Purpose: Refers to the current object instance in a method or constructor.
  • Use Cases:
    • Distinguishes between instance variables and parameters with the same name.
    class Student {
		int rollno;
		String name;
		Student(int rollno, String name) {
			this.rollno = rollno;
			this.name = name;
		}
	}
  • Calling Constructors Using this():
    • Calls another constructor in the same class.
    class A {
		A() { System.out.println("Hello A"); }
		A(int x) { this(); System.out.println(x); }
	}

7. Overloaded Constructors

  • Definition: Multiple constructors in the same class with different parameter lists.
    class Demo {
		int value1, value2;
		Demo() { value1 = 10; value2 = 20; }
		Demo(int a) { value1 = a; }
		Demo(int a, int b) { value1 = a; value2 = b; }
	}

8. Default and No-Argument Constructors

  • Default Constructor: Created automatically by the Java compiler if no constructor is defined.
  • No-Argument Constructor: Explicitly defined constructor with no parameters.
    class MyClass {
		int x;
		MyClass() { x = 0; } // No-Argument Constructor
	}

9. Composition

  • Definition: A class can include objects of other classes as members.
    class AlarmClock {
		Time currentTime;
		Time alarmTime;
	}

10. enum Types

  • Definition: Defines a set of constants as unique identifiers.
    public enum Level { LOW, MEDIUM, HIGH }
	Level myVar = Level.MEDIUM;
  • Traversing Enum:
    for (Level l : Level.values()) System.out.println(l);

11. Garbage Collection

  • Purpose: Automatically reclaims unused memory in Java.
  • Finalizer Method: Cleans up resources before an object is garbage collected.
    protected void finalize() { System.out.println("Garbage Collected"); }

12. Static Class Members & Static Import

  • Static Members: Shared across all objects of a class.
    class Calculate {
		static int cube(int x) { return x * x * x; }
	}
  • Static Import: Allows importing static members for direct access.
    import static java.lang.Math.*;
	System.out.println(sqrt(16));

13. Final Instance Variables

  • Definition: Declares a variable as constant (cannot be modified).
    class Bike {
		final int speedLimit = 90;
	}

14. Creating Packages

  • Purpose: Groups related classes and interfaces to manage complexity and promote reuse.
    package mypack;
	public class MyClass { public void msg() { System.out.println("Hello"); } }
